# v1.0 可视化路线图

这份路线图用于回答一个问题：v1.0 现在到底准备怎么做。

## 1. 最终定位

```mermaid
flowchart LR
  V10["v1.0<br/>Modular Document Kernel"]
  Core["Core<br/>必装"]
  Packs["Optional Capability Packs<br/>可选能力包"]
  CLI["CLI<br/>check / status / init / add-capability"]
  Dash["Basic HTML Dashboard<br/>只读展示"]
  Verify["Verification Contract<br/>模板不变量 + 任务填槽"]
  Adopt["Safe Adoption<br/>旧项目安全接入"]

  V10 --> Core
  V10 --> Packs
  V10 --> CLI
  V10 --> Dash
  V10 --> Verify
  V10 --> Adopt
```

一句话：v1.0 是一个可安装、可检查、可裁剪、可视化查看状态的文档内核。

## 3. Scaffold / Configure 分层

```mermaid
flowchart TB
  Diagnose["1 Diagnose<br/>扫描项目"]
  Decide["2 Decide<br/>选 delivery model + capability packs"]
  Scaffold["3 Scaffold<br/>CLI 机械创建结构"]
  Configure["4 Configure<br/>Agent 对话式定制"]
  Verify["5 Verify<br/>harness check"]
  Deliver["6 Deliver<br/>交付 summary + 首批任务"]

  Diagnose --> Decide --> Scaffold --> Configure --> Verify --> Deliver
```

关键原则：

| 层 | 谁做 | 做什么 | 不做什么 |
| --- | --- | --- | --- |
| Scaffold | CLI | 目录、模板、空表、`.harness-capabilities.json` | 不生成假 reference |
| Configure | Agent + 用户 | AGENTS、references、CI、regression surface | 不机械复制源仓标准 |

## 4. 四个实现 Wave

```mermaid
flowchart LR
  W0["Wave 0<br/>合同收敛"]
  W1["Wave 1<br/>CLI 内核"]
  W2["Wave 2<br/>内容与示例"]
  W3["Wave 3<br/>文档与发布"]

  W0 --> W1 --> W2 --> W3
```

| Wave | 目标 | 包含 |
| --- | --- | --- |
| Wave 0 | 锁定范围 | scope、non-goals、capability packs、P2 关闭 |
| Wave 1 | CLI 内核 | package、capability registry、check、status、init、add-capability |
| Wave 2 | 内容与示例 | SKILL.md 六阶段、minimal/full examples、basic dashboard、safe adoption |
| Wave 3 | 发布收口 | public docs、CI、verification contract、final review |

## 6. 只读 Dashboard

```mermaid
flowchart LR
  MD["Harness Markdown"]
  Status["harness status --json"]
  HTML["single-page HTML dashboard"]
  Human["Human review"]

  MD --> Status --> HTML --> Human
  HTML -. no writes .-> MD
```

Dashboard 只展示：

- 当前任务进度
- installed capabilities
- pending handoff
- open risk
- recent activity

不做控制，不写文件，不做后端。

## 7. 旧项目 Safe Adoption

```mermaid
flowchart TB
  Old["Old harness project"]
  Fallback["check/status fallback file detection"]
  Suggest["suggest .harness-capabilities.json"]
  Human["coordinator/human reviews"]
  Write["explicit adoption write"]
  Check["harness check"]

  Old --> Fallback --> Suggest --> Human --> Write --> Check
  Suggest -. no auto apply .-> Old
```

v1.0 支持安全接入，不支持自动迁移：

- 不重装旧项目。
- 不覆盖旧事实。
- 不自动改 AGENTS / registry / ledger。
- 不自动 archive 历史目录。
- 只给 adoption suggestion，由 coordinator/human 确认后写入。

## 8. 执行策略记录

```mermaid
flowchart LR
  Task["task_plan.md"]
  Strategy["Execution Strategy"]
  Subagents["subagent mode"]
  Review["review model"]
  Evidence["evidence depth"]

  Task --> Strategy
  Strategy --> Subagents
  Strategy --> Review
  Strategy --> Evidence
```

每个任务都要记录：

- 主执行者是谁。
- 是否用 subagent。
- subagent 是 reviewer-only 还是 worker-worktree。
- review 模型是什么。
- 证据深度是多少。
- worktree / handoff 怎么处理。

## 9. 发布门禁

```mermaid
stateDiagram-v2
  [*] --> Planned
  Planned --> Implementing
  Implementing --> Checked
  Checked --> Verified
  Verified --> ReleaseCandidate
  ReleaseCandidate --> ReadyToTag
  ReleaseCandidate --> Blocked
  Blocked --> Implementing
```

Release candidate 前必须满足：

- 无 open P0/P1/P2 strategy findings。
- `scripts/check-harness.mjs` 旧入口不坏。
- `.harness-capabilities.json` 支持 dependency + state validation。
- checker 使用静态规则组，不动态加载代码。
- `harness status` 只读。
- dashboard 只读。
- examples 通过 checker。
- CI 只跑已实现命令。
- verification contract 有结构化输出。
- 每个非平凡任务记录 execution strategy。
- 旧项目 safe adoption 不自动 apply。
